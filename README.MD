Краткий гайд по тому, как работать над проектом.

---

### 1. Подготовка (один раз)
Клонируйте репозиторий:
```bash
git clone https://github.com/CodeForgeHQ/BiteDash.git
cd BiteDash
```

---

### 2. Работа над новой задачей
**Всегда** создавайте новую ветку для своей задачи:
```bash
# Обновите main, чтобы быть в курсе последних изменений
git checkout main
git pull origin main

# Создайте свою ветку (назовите по смыслу, например feature/login-page)
git checkout -b feature/название-вашей-фичи
```

---

### 3. Сохранение и отправка кода
Когда вы написали код и готовы его показать:
```bash
# Добавьте изменения
git add .

# Сделайте коммит (кратко опишите, что сделано)
git commit -m "добавил форму входа"

# Отправьте ветку в репозиторий
git push origin feature/название-вашей-фичи
```

---

### 4. Финальный шаг
1. Перейдите на [GitHub BiteDash](https://github.com/CodeForgeHQ/BiteDash)
2. Нажмите кнопку **"Compare & pull request"**
3. Ссылку на созданный PR отправьте в рабочую группу.

### Про именования коммитов

Все коммиты должны быть понятными и описывать, что было сделано.

Примеры:
- "fix: исправил баг с отображением списка заказов"
- "feat: добавил возможность добавления новых ресторанов"
- "docs: добавил описание архитектуры проекта"
- "refactor: улучшил производительность при загрузке данных"
- "test: добавил тесты для компонентов"
- "chore: обновил зависимости проекта"
- "perf: улучшил производительность при загрузке данных"
- "ci: настроил CI/CD"
- "build: настроил сборку проекта"
- "release: выпустил новую версию проекта"
- "hotfix: исправил критический баг в продакшн-версии"
- "wip: работа над новой фичей"
- "skip: пропустил тесты"
